# 公开评论采集流程

## 目标

尽可能收集某个产品在多个电商平台上的公开可见用户评价，并整理成可分析的数据集。

## 先判断采集方式

按优先级选择：

1. 用户已提供评论导出文件：直接标准化
2. 用户提供商品链接：逐个平台查看公开评论并整理
3. 用户只提供产品名或图片：先定位同款/相似款链接，再整理评论
4. 用户要求最新评论但页面受限：请求授权使用登录态，或请用户导出文件

## 平台可见性判断

采集前先确认：

- 评论是否公开可见
- 是否需要登录才能翻更多页
- 是否存在地区限制
- 是否有明显验证码或反爬
- 是否能稳定看到评分、评论正文、时间、变体

如果评论只显示部分样本，要在结论里注明“采样自公开可见评论页，不代表平台全部评论”。

## 最低采集字段

每个平台至少尝试收集：

- `platform`
- `product_id` 或商品链接
- `product_title`
- `brand`
- `rating`
- `review_title`
- `review_text`
- `review_date`
- `variant`
- `country`
- `verified_purchase`
- `helpful_votes`
- `review_url`

缺字段可以继续，但要记录缺失。

## 采集顺序建议

如果用户要求“全球平台评价”，默认优先：

1. Amazon
2. Walmart
3. eBay
4. Etsy
5. AliExpress
6. Temu
7. TikTok Shop

再根据品类补充：

- 美妆个护：iHerb、Sephora、Ulta
- 家具家居：Wayfair、Target
- 宠物用品：Chewy
- 母婴：Target、Walmart、Amazon

## 采集方式建议

### A. 用户给导出文件

最稳妥。直接运行：

```bash
./scripts/normalize_marketplace_reviews.py raw-reviews.csv -o normalized-reviews.csv
./scripts/analyze_reviews.py normalized-reviews.csv -o review-analysis.md
```

### B. 用户给商品链接

逐个页面整理，形成统一 CSV。优先抓：

- 评分
- 评论数
- 近 10 到 50 条代表性评论
- 近期差评
- 图片评论
- 变体相关评论

### C. 需要授权登录态

只有在用户明确授权后，才可以使用浏览器 cookies 或登录态去读取更多公开评论页。

## 真实评价的表达方式

不要说“拿到了全部真实评价”。应该说：

- 采集了平台公开可见评论
- 识别了重复出现的用户反馈
- 分析了高频满意点和痛点
- 标出了可疑评论信号

## 输出模板

```markdown
# 公开评论采集说明

## 产品与平台范围

## 采集来源

## 平台覆盖情况

| 平台 | 链接 | 评论是否公开 | 采集方式 | 样本量 | 限制 |
|---|---|---|---|---:|---|

## 字段覆盖情况

## 数据缺口

## 后续分析建议
```

## 风险提醒

- 评论样本可能受地区和登录态影响
- 平台展示顺序可能偏向“最相关”而不是“全部”
- 同款不同链接、变体合并和翻新商品会影响判断
- 评论时间跨度过长时，旧评论可能不代表当前版本
