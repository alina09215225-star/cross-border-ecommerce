# 跨平台真实评价分析

## 目标

从真实用户评价里提炼：

- 用户为什么买
- 用户为什么满意
- 用户为什么退货或差评
- 产品该怎么改
- 页面、广告、短视频该强调什么

分析前，先区分：

- `公开评论采集`：见 [public-review-collection.md](./public-review-collection.md)
- `评论分析`：在拿到评论样本后，做主题和洞察提炼

## 评论字段

优先收集：

- `platform`
- `product_id`
- `product_title`
- `brand`
- `rating`
- `review_title`
- `review_text`
- `review_date`
- `variant`
- `country`
- `verified_purchase`
- `helpful_votes`
- `review_url`

缺字段也可以分析，但要说明置信度降低。

如果评论文件来自不同平台、字段混乱，先运行：

```bash
./scripts/normalize_marketplace_reviews.py raw-reviews.csv -o normalized-reviews.csv
```

## 清洗规则

处理评论时：

- 合并标题和正文
- 去除空评论、重复评论、明显模板化评论
- 保留原始评分和日期
- 把变体、尺寸、颜色等信息保留在主题分析里
- 对机器翻译评论要标注可能有语义损失

## 主题分类

默认主题：

- 质量和耐用性
- 材质和做工
- 尺寸和适配
- 易用性
- 安装和清洁
- 功能效果
- 设计和外观
- 包装和物流
- 售后和退换
- 价格和性价比
- 气味、安全、合规

根据品类补充专用主题。例如：

- 家居收纳：承重、稳定性、空间利用
- 宠物用品：宠物接受度、噪音、安全
- 美妆个护：刺激、肤感、气味、效果周期
- 厨房用品：清洁难度、食品接触安全、容量

## 判断真实痛点

高价值痛点通常满足：

- 不同平台反复出现
- 高评分和低评分都提到，只是角度不同
- 影响退货、差评或复购
- 能通过产品设计、包装、说明书或详情页改进

低价值噪音通常是：

- 单个极端个案
- 物流偶发
- 用户误用
- 与产品定位不匹配的期望

## 可疑评论信号

不要直接定性“刷评”，只说“可疑信号”：

- 大量短时间五星短评
- 文案高度相似
- 没有具体使用场景
- 评分极高但图片/细节很少
- 评论时间与上架/促销异常集中
- 不同账号重复相同措辞
