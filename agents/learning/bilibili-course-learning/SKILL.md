# Bilibili 课程学习技能包

这个技能用于把 B 站课程从“看过”变成“学会并能用”。它的核心不是单纯总结，而是按“找课/接收课程 -> 提取文本 -> 搭建理解 -> 主动复述 -> 迁移应用”的流程，把课程变成可执行结果。

优先处理四类任务：

1. 用户给出 B 站链接，希望提炼课程内容
2. 用户给出字幕、转写稿或笔记，希望重组为更适合学习的资料
3. 用户已经学过一门课，希望做复盘、问答、练习或应用方案
4. 用户希望把课程内容迁移到自己的项目、岗位、产品或业务情境
5. 用户要求主动去 B 站筛选一批课程，例如“找 10 门跨境电商好课并总结”

## 快速决策

先判断用户任务属于哪一类：

- `单课学习`：用户给出一个 B 站链接，直接提取文本并总结
- `多课筛选`：用户要求找课、选课、比较课程，先读 [course-discovery.md](./references/course-discovery.md)
- `已有文本整理`：用户给了字幕、文稿、笔记，直接清洗和学习加工
- `环境设置`：用户要把技能装到新电脑，先读 [setup.md](./references/setup.md)

如果用户说“好评率最高”，不要假装 B 站有统一好评率。应改用播放、点赞、收藏、投币、弹幕/评论质量、课程完整度、发布时间和 UP 主可信度作为代理指标，并明确说明这是“筛选指标”，不是平台官方好评率。

## 先判断输入材料

按下面顺序选择最省力的内容来源：

1. 如果用户已经给了课程字幕、文稿、笔记或 markdown，直接使用，不要重复提取
2. 如果用户给了 B 站链接，优先用 `./scripts/fetch_bilibili_course_material.py` 提取现成字幕；拿不到字幕时，自动退到本地音频转写并生成学习稿
3. 如果只有 `.srt`、`.vtt`、B 站字幕 JSON，用 `./scripts/normalize_bilibili_subtitles.py` 清洗成纯文本
4. 如果视频很长，先用 `./scripts/prepare_bilibili_study_material.py` 生成分块学习稿，再继续分析

如果本地没有下载或转写依赖，不要硬编造内容。此时应明确告知用户需要：

- 提供字幕/转写文本
- 或先用浏览器脚本复制字幕
- 或补装下载/转写工具后再继续

链接直提脚本依赖：

- `yt-dlp`
- 如果要走后续转写 fallback，再补 `ffmpeg`
- `whisper-cpp`
- 一个本地 `ggml` Whisper 模型文件；默认优先使用 `ggml-small.bin`，再退回 `ggml-base.bin`

新电脑首次使用时，优先运行：

```bash
./scripts/setup_macos_runtime.sh
./scripts/download_whisper_model.sh small
./scripts/check_environment.py
```

## 核心工作流

真正执行学习任务时，固定按下面 5 步走。

### 1. 明确学习目标

先识别用户最想得到哪种结果，只选一个主目标驱动本轮输出：

- `快速理解`：先给摘要、框架、关键词
- `系统学习`：输出分层笔记、概念解释、章节地图
- `准备复习`：输出问答卡、易错点、速记版
- `准备应用`：输出行动清单、方案草稿、案例迁移
- `准备分享/教学`：输出讲解提纲、复述稿、演示结构

如果用户没说明，就默认目标是：`系统学习 + 准备应用`。

### 2. 把课程拆成“知识单元”

不要只顺着视频时间线写流水账，而是把内容重组为知识单元。每个单元尽量包含：

- 这个单元在解决什么问题
- 关键概念或方法
- 原视频里的例子
- 这个方法适用的边界
- 用户自己的业务/项目里可怎么用

### 3. 产出学习材料

至少输出下面 4 个部分：

1. `一句话总结`：这门课最重要的结论
2. `结构化笔记`：按知识单元整理
3. `主动回忆题`：帮助用户不用看原文也能回忆
4. `应用迁移`：把方法放到用户真实场景

如果内容足够多，再补：

- `术语表`
- `反例 / 易错点`
- `执行 checklist`
- `7 天复习计划`

### 4. 强制做“输出型学习”

仅有总结还不够。每次都至少追加一种输出型环节：

- 费曼解释：用更白话的方式重讲一遍
- 问答模式：生成 5 到 10 个问题并自答
- 案例迁移：换到用户行业、团队、产品再推演一次
- 动作设计：列出下一步能马上做的 3 到 5 个动作

### 5. 落地到应用

当用户问“怎么用”时，不要泛泛而谈，直接映射到真实场景：

- 如果是产品/运营课：产出策略、实验、指标和复盘框架
- 如果是销售/商业课：产出话术、价值主张、客户分层或转化动作
- 如果是管理/组织课：产出会议机制、决策模板、沟通动作
- 如果是技能教程：产出实践步骤、最小 demo、常见坑

## 多课检索与学习

当用户要求“去 B 站找 N 门课再学习总结”时：

1. 先拆关键词，不要只搜一个词。例如跨境电商要覆盖 `跨境电商`、`亚马逊运营`、`TikTok Shop`、`独立站`、`Shopify`、`出海营销`、`选品`、`广告投放`
2. 先给出候选池，再做排序；候选池通常应大于最终数量的 2 到 3 倍
3. 评分要解释依据，不要只给结论
4. 最终学习时，优先处理排序最高、课程完整度最高、和用户目标最贴近的视频
5. 如果用户要 10 门深度总结，建议先交付 `10 门课程清单 + 推荐学习顺序 + 前 3 门深度总结`，再继续批量处理

详细筛选规则见 [course-discovery.md](./references/course-discovery.md)。

## 输出模板

默认优先使用下面这个结构：

```markdown
# 课程学习结果

## 一句话结论

## 这门课到底在讲什么

## 结构化笔记

## 关键概念 / 方法

## 主动回忆题

## 如何用到你的场景

## 下一步行动
```

如果用户明确说要更短，就压缩成：`摘要 + 3 个关键点 + 3 个行动`。

## 长视频处理

如果课程很长或是系列课，不要一次性把所有内容混在一起。优先分段：

1. 先生成章节或分块目录
2. 每块单独提炼重点
3. 最后再做总整合

`./scripts/prepare_bilibili_study_material.py` 可以把原始文本切成适合后续分析的分块学习稿。

## 一键提取建议

如果本机已安装 `yt-dlp`，优先直接运行：

```bash
./scripts/fetch_bilibili_course_material.py "https://www.bilibili.com/video/BVxxxxx" -o ./study-output
```

如果用户已授权读取浏览器登录态，可加：

```bash
./scripts/fetch_bilibili_course_material.py "https://www.bilibili.com/video/BVxxxxx" -o ./study-output --cookies-from-browser chrome
```

它会尽量：

1. 读取视频标题与字幕语言
2. 优先选中文人工字幕，其次中文自动字幕
3. 如果没有字幕，自动回退到本地音频转写
4. 输出纯文本转写
5. 输出可继续学习的 markdown 学习稿

## 何时不要使用这个技能

下面这些情况不要硬套本技能：

- 用户只想下载视频，不关心学习整理
- 用户没有任何可提取文本，也不允许补装工具或提供字幕
- 用户的问题与课程内容无关，只是普通闲聊
- 用户需要法律、医疗、投资等高风险专业建议
