# 场景迁移

## 原则

迁移不是“举个例子”，而是把课程中的判断改写成用户的真实决策。

## HR / 组织场景

优先迁移到：

- 招聘判断标准
- 面试追问逻辑
- 管理动作设计
- 组织协作机制
- 培养与反馈机制

## 课程运营 / 班级服务场景

优先迁移到：

- 学员分层
- 服务动作
- 活动节奏
- 反馈闭环
- 转化节点

## 招生 / 销售场景

优先迁移到：

- 用户需求判断
- 内容表达
- 话术设计
- 跟进优先级
- 转化实验

## AI 落地场景

优先迁移到：

- 哪些任务适合流程化
- 哪些判断适合 agent 辅助
- 哪些材料值得沉淀成 skill
- 哪些步骤需要人保留最终判断

## 迁移输出模板

```markdown
# 场景迁移

## 原课判断

## 对你的场景意味着什么

## 哪个动作最该先改

## 如果照旧不改，会继续出现什么问题

## 最小验证动作
```
