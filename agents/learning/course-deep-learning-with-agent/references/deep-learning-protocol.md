# 深度学习协议

## 目标

让用户不是“拿到一个正确答案”，而是“形成自己的判断能力”。

## 基本顺序

### 1. 定义学习问题

先明确：

- 这门内容要解决什么问题
- 用户最想学懂什么
- 用户最终需要的结果是什么

### 2. 抽出主判断

不要同时给很多观点。先找出课程最核心的 1 到 3 个判断。

### 3. 展开判断链

每个判断都按下面顺序解释：

1. 结论是什么
2. 为什么会这样
3. 原文有什么证据
4. 什么情况会失效
5. 放到用户场景里怎么理解

### 4. 处理误解

对每个关键判断，都补一组：

- 常见误解
- 正确理解
- 为什么容易误解

### 5. 形成用户自己的语言

真正学会的标志不是能复述原文，而是能用自己的话说出来。要帮助用户生成：

- 一句话复述
- 面向同事的讲法
- 面向业务场景的讲法

### 6. 最后再行动

只有在用户已经看懂核心判断后，再生成：

- 行动方案
- 检查清单
- 下一步实验

## 判断是否“学透了”

至少满足 4 条中的 3 条：

1. 能说出课程在回答什么问题
2. 能解释为什么这样判断
3. 能说出适用边界或反例
4. 能放到自己的场景里推演一遍
