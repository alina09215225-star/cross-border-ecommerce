# 理解检验

## 检验原则

不要只考记忆。优先考：

- 是否理解因果
- 是否识别边界
- 是否能迁移
- 是否能区分错误理解

## 默认五问

1. 这门课到底在回答什么问题
2. 它最核心的判断是什么
3. 这个判断为什么成立
4. 在什么情况下它不成立
5. 如果放到你的业务里，第一步会怎么做

## 追问模板

- 如果把前提条件拿掉，这个结论还成立吗
- 有没有和它相反但也合理的情况
- 这门课最容易被误用在哪里
- 你现在准备怎么向别人解释它
- 你打算如何验证自己不是“听起来懂了”

## 输出格式

```markdown
# 理解检验

## 判断题 / 追问题

## 标准答案

## 容易误解的地方

## 如果没答出来，应该回看哪一层
```
