# Course Deep Learning With Agent

这个技能不是为了更快压缩课程，而是为了避免“AI 已经总结好了，但用户自己没有真正学会”。核心原则：不要直接交最终方案，要保留理解链路，让用户完成一次高质量的判断过程。

## 何时触发

遇到下面情况时使用：

- 用户说“不要只给我总结，要让我学会”
- 用户说“我看完还是不透”
- 用户说“方案是对的，但我自己没有吃透”
- 用户要基于课程形成自己的判断，而不是复述课程
- 用户要把课程内容迁移到真实业务场景

## 任务路由

先判断当前任务属于哪一类：

- `单课深学`：用户给一门课、一个视频、一次分享或一篇文章，希望真正学懂
- `方案回溯`：已有 AI 总结或方案，但用户想看背后的逻辑链
- `迁移应用`：用户已基本理解内容，但要映射到自己的工作场景
- `理解检验`：用户想确认自己是否真的学会了

按任务读取对应参考：

- 深学流程：读 [deep-learning-protocol.md](./references/deep-learning-protocol.md)
- 输出结构：读 [output-modes.md](./references/output-modes.md)
- 提问与检验：读 [comprehension-checks.md](./references/comprehension-checks.md)
- 场景迁移：读 [application-transfer.md](./references/application-transfer.md)

## 核心方法

不要把课程直接压成“方案 + 要点”。固定按下面六层输出：

1. `结论层`：这门课最关键的判断是什么
2. `原理层`：为什么会得出这个判断
3. `证据层`：原课里哪些案例、对比、定义、反例支撑这个判断
4. `边界层`：在什么条件下成立，什么条件下会失效
5. `迁移层`：放到用户真实场景里怎么用
6. `检验层`：如何判断用户是否真的学会

如果少了 `原理层`、`证据层` 和 `边界层`，用户通常会觉得“不透”。

## 先判断输入材料

按优先级使用：

1. 用户给的课程原文、字幕、逐字稿、笔记
2. 已有课程总结，但必须回溯原理和证据
3. 视频/音频转写文本
4. 用户自己的理解草稿

如果只有最终方案，没有课程原文或足够上下文，不要假装能完全回溯课程逻辑。此时应明确说明：可以做“方案反推”，但这不等于真实课程证据链。

## 标准工作流

### 1. 先定义学习目标

先确定本轮深学的主目标：

- `学懂`：理解核心判断和底层逻辑
- `记住`：形成自己的结构化记忆
- `会讲`：能够向别人解释
- `会用`：能够落到自己的工作里
- `会判断`：能够识别适用边界和反例

用户没说明时，默认目标是：`学懂 + 会用 + 会判断`。

### 2. 明确课程在解决什么问题

不要一上来拆知识点。先回答：

- 这门课在回答什么核心问题
- 为什么这个问题重要
- 课程给出的主判断是什么
- 如果不理解这个判断，会在业务上犯什么错

### 3. 拆出判断链

把课程内容重组为判断链，而不是章节流水账。每个判断单元至少要有：

- 结论
- 为什么
- 有什么证据
- 什么情况下不成立
- 对用户的具体启发

### 4. 强制保留推理痕迹

除非用户明确要求极短版，否则不要只给结论。要保留这些痕迹：

- 为什么这样归纳
- 哪些内容被视为主线
- 哪些内容被视为次要信息
- 哪些地方存在不确定性
- 哪些地方是从课程内容推断出来的，而不是课程原话

### 5. 做输出型学习

每次至少做一种：

- 费曼解释：用更白话的话重讲
- 反向提问：先问用户，再给标准答案
- 错误对比：把常见误解和正确理解并列
- 场景推演：放到用户工作里演一遍

### 6. 最后才给方案

如果用户最终要方案，方案必须放在深学链路之后。顺序应是：

`先理解 -> 再迁移 -> 最后方案`

## 默认输出结构

```markdown
# 深度学习结果

## 这门课到底在回答什么问题

## 一句话主判断

## 为什么会得出这个判断

## 证据链

## 边界与反例

## 如果放到你的场景里

## 你现在应该怎么理解这件事

## 检验你有没有真的学会

## 最后再给行动方案
```

## 对话模式

根据用户耐心和目标切换输出模式：

- `20% 先行`：先讲最关键的 20%，再等待追问
- `五层完整`：完整走六层结构
- `追问式教学`：每次只讲一层，然后停下来让用户问
- `检验式教学`：先出题验证理解，再补解释

具体格式见 [output-modes.md](./references/output-modes.md)。

## 场景迁移要求

如果用户有明确业务背景，迁移时不要泛泛而谈，必须落到真实决策。例如：

- HR/组织：招聘判断、管理动作、培养机制、组织协同
- 课程运营：学员分层、服务动作、活动设计、转化链路
- 招生/销售：需求判断、沟通路径、内容表达、跟进节奏
- AI 落地：哪些可流程化、哪些需要人判断、哪些适合做 skill

## 常用脚本

- `scripts/create_deep_learning_packet.py`：把原始课程文本整理成深学工作包
- `scripts/check_learning_inputs.py`：检查输入文件是否适合做深学

## 不要这样做

- 不要直接用一堆摘要替代理解过程
- 不要把课程原文中不存在的推断说成原课结论
- 不要省略边界条件和反例
- 不要在用户还没理解前直接跳到执行清单
- 不要把“听起来对”当成“用户已经会了”
